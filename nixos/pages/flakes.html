❄️ Flakes

¿Qué son los Flakes? Son la forma moderna de gestionar tu configuración de NixOS. Imagina que tu configuración es un "proyecto" con sus propias dependencias versionadas, como un proyecto de programación con su package.json.

El archivo flake.nix declara qué usar y el flake.lock "congela" las versiones exactas para que la compilación sea siempre idéntica. Esto hace que tu sistema sea reproducible: si copias los archivos a otro ordenador, obtienes exactamente lo mismo.

Canales vs Flakes: Los canales son como un "apt update" global (simple pero menos control). Los flakes son proyectos independientes con dependencias versionadas (más control, más reproducible).

⚠️ Configuración necesaria y error común

Habilitar flakes: Para usar flakes necesitas añadir esta línea en tu configuration.nix:

nix.settings.experimental-features = [ "nix-command" "flakes" ];

Sin esto, los comandos de flakes no funcionarán.

Error común durante la migración:
Aún después de añadir la línea anterior, puedes encontrar este error:
error: experimental Nix feature 'nix-command' is disabled; add '--extra-experimental-features nix-command' to enable it

Causa: El daemon de Nix no tiene habilitados los features experimentales hasta que se reinicie (lo que ocurre al aplicar la nueva configuración con flakes).

Solución temporal: Usa el flag --extra-experimental-features:

cd /etc/nixos && sudo nix --extra-experimental-features 'nix-command flakes' flake update

Una vez que se aplique la configuración con flakes, el error desaparecerá porque el daemon se reiniciará con los features habilitados.

Flujo de actualización del sistema

Proceso completo: Actualizar paquetes con flakes requiere dos pasos: 1) Actualizar las versiones en el flake, 2) Aplicar los cambios al sistema.

Recomendación: Siempre prueba primero con dry-build para ver qué cambiará.

Opción 1: Actualizar TODO (más arriesgado)Actualiza tanto nixpkgs estable como unstable. Puede cambiar muchas cosas en el sistema.
Para obtener las últimas versiones de todo, incluyendo el núcleo del sistema.
● ● ● bash 
cd /etc/nixos && sudo nix --extra-experimental-features 'nix-command flakes' flake update
cd /etc/nixos && sudo nixos-rebuild switch --flake .#mi-hostname
Opción 2: Actualizar solo paquetes unstable (recomendado)Actualiza solo los paquetes marcados como 'unstable' en tu configuración. Más seguro.
Para obtener versiones nuevas de aplicaciones (Brave, Neovim, etc.) sin tocar el núcleo del sistema.
● ● ● bash 
cd /etc/nixos && sudo nix --extra-experimental-features 'nix-command flakes' flake lock --update-input nixpkgs-unstable
cd /etc/nixos && sudo nixos-rebuild switch --flake .#mi-hostname
Prueba antes de aplicarVer qué cambiará sin modificar el sistema.
Simula la reconstrucción para ver los paquetes que se actualizarán.
● ● ● bash 
cd /etc/nixos && sudo nixos-rebuild dry-build --flake .#mi-hostname

¿Qué significan los inputs?

  • nixpkgs → Versión estable de NixOS (nixos-25.11). Cambiarlo actualiza el núcleo del sistema.
  • nixpkgs-unstable → Versión de desarrollo. Solo afecta a los paquetes que usan unstable. en tu configuración.

Frecuencia recomendada: Actualiza nixpkgs-unstable semanalmente para tener aplicaciones actualizadas. Actualiza nixpkgs solo cuando necesites nuevas características del sistema.

Migración desde canales a flakes

Proceso real ejecutado: Estos son los comandos exactos que se usaron para migrar este sistema de canales tradicionales a flakes. La migración es segura y reversible si algo sale mal.

Características de este proceso:

  • Incluye solución al error común experimental Nix feature 'nix-command' is disabled
  • Usa rutas absolutas en los comandos (sin necesidad de cd)
  • Muestra la salida real obtenida durante la migración
  • Compatible con hostname mi-hostname y usuario usuario
Paso 1: Copiar archivos de configuraciónPrepara los archivos flake.nix y configuration.nix en /etc/nixos/
Copia los archivos desde la carpeta de migración al directorio de configuración del sistema.
● ● ● bash 
sudo cp -i ~/migracion_a_flakes/flake.nix ~/migracion_a_flakes/configuration.nix /etc/nixos/
El flag -i pregunta antes de sobrescribir si ya existe configuration.nix.
Paso 2: Generar flake.lock (con solución a error común)Actualiza los inputs y crea el archivo de versiones exactas.
Este paso puede fallar con el error de features experimentales deshabilitados. La solución es añadir --extra-experimental-features.
● ● ● bash 
cd /etc/nixos && sudo nix --extra-experimental-features 'nix-command flakes' flake update  # Solución al error 'experimental Nix feature...'

¿Por qué usar cd primero?

Los comandos nix flake update y nix flake lock buscan flake.nix desde el directorio actual. Usar cd /etc/nixos && ... evita errores de "could not find a flake.nix file". nixos-rebuild sí acepta rutas absolutas, pero para consistencia es mejor usar cd primero en todos los comandos.

¿Qué contiene cada archivo?

  • flake.nix → Define las dependencias (inputs) y cómo construir el sistema (outputs). Es el "contrato" principal: especifica qué versiones de nixpkgs usar y qué configuración generar (nixosConfigurations.mi-hostname).
  • configuration.nix → Configuración completa del sistema. Recibe dos parámetros: pkgs (paquetes estables) y unstable (paquetes inestables). Todo lo que defines aquí (usuario, servicios, kernel, etc.) se aplica al sistema.
  • flake.lock → Archivo generado automáticamente que "congela" las versiones exactas de los inputs. Garantiza que siempre se compile lo mismo (reproducibilidad). No se edita manualmente; se regenera con nix flake update.

Flujo: flake.nix → (genera) → flake.lock → (usa) → configuration.nix → (produce) → sistema NixOS.

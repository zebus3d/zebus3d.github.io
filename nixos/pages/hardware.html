🖥️ Hardware y Drivers

Configurar hardware en NixOS es 100% declarativo. Todo va en configuration.nix: drivers de GPU, virtualización, audio, fuentes, y apps que necesitan librerías del sistema.

NVIDIA — GPU dual (Prime)

hardware.opengl ▶ hardware.graphics: Desde NixOS 24.11 (diciembre 2024), hardware.opengl fue renombrado a hardware.graphics. Si tu config aún usa opengl, cámbialo.

Configuración NVIDIA PrimeActiva gráfica NVIDIA para rendering y configura soporte para Prime (dos GPUs).Para portátiles con NVIDIA+Intel
GPU dual config.
● ● ● bash 
# OpenGL
hardware.graphics = {
    enable = true;
    enable32Bit = true;
};

# NVIDIA driver
services.xserver.videoDrivers = [ "nvidia" ];
hardware.nvidia = {
    modesetting.enable = true;
    open = false;
    package = config.boot.kernelPackages.nvidiaPackages.stable;
    prime = {
        sync.enable = true;
        intelBusId = "PCI:0:2:0";
        nvidiaBusId = "PCI:1:2:0";
    };
};

Sync vs Offload: sync = rendimiento máximoAmbas GPUs siempre activas. Ideal para escritorio o si no te importa la batería. — para portátiles con batería, usa offload.enable = true y lanza apps con __NV_PRIME_RENDER_OFFLOAD=1.

Cómo obtener tus Bus IDs: Ejecuta lspci | grep -E "VGA|3D" para ver tus GPUs. Luego convierte los números de hexadecimal a decimal (ej: 01:00.0 → PCI:1:0:0, 05:00.0 → PCI:5:0:0).

Actualizar kernel con NVIDIA: En NixOS, los drivers NVIDIA (config.boot.kernelPackages.nvidiaPackages.stable) se recompilan automáticamente para el kernel actual. Siempre prueba primero:

● ● ● bash 
sudo nixos-rebuild dry-build --flake /etc/nixos

Si compila sin errores → es seguro actualizar. Si falla → kernel/driver no compatibles.

Estrategias seguras:

  • Actualizar solo paquetes unstable (mantiene kernel estable):
    cd /etc/nixos && sudo nix --extra-experimental-features 'nix-command flakes' flake lock --update-input nixpkgs-unstable
  • Actualizar kernel (nixpkgs estable) cada 2-3 meses después de verificar con dry-build
  • Usar generaciones como red de seguridad: sudo nixos-rebuild switch --rollback si falla

Reiniciar: Después de actualizar el kernel, reinicia para usar la nueva versión. Verifica con uname -r.

nix-ld — Apps externas

¿Qué es nix-ld? Permite ejecutar binarios que no fueron compilados para NixOS (como Blender descargado de la web). Proporciona las librerías del sistema que estos programas esperan.

Configurar nix-ldHabilita el wrapper global para binarios externos.
Binarios no NixOS.
● ● ● bash 
programs.nix-ld = {
    enable = true;
    libraries = with pkgs; [
        libx11 libxext libxrender
        libglvnd fontconfig freetype
        libjpeg_turbo libpng zlib
        libxkbcommon wayland
        # Para Blender con CUDA:
        linuxPackages.nvidia_x11
        cudaPackages.cuda_cudart
    ];
};

¿Cuándo usar nix-ld? Cuando descargas un binario precompilado de su web oficial (Blender, OBS, Spotify, Discord, Steam).

Blender binario precompilado

Blender descargado de la web requiere configuración extra para Vulkan y OpenGL en NixOS.

Wrapper para BlenderScript que busca librerías dinámicamente y configura Vulkan/OpenGL.
launcher.
● ● ● bash 
#!/usr/bin/env bash
# Blender launcher script for NixOS
BLENDER_PATH="$1"
shift

if [[ -z "$BLENDER_PATH" ]]; then
    echo "Error: No blender path provided"
    exit 1
fi

# Soporte para NixOS
[ -n "$NIX_LD_LIBRARY_PATH" ] && export LD_LIBRARY_PATH="$NIX_LD_LIBRARY_PATH:$LD_LIBRARY_PATH"
export VK_ICD_FILENAMES=/run/opengl-driver/share/vulkan/icd.d/nvidia_icd.x86_64.json
export LD_LIBRARY_PATH=/run/opengl-driver/lib:$LD_LIBRARY_PATH

exec "$BLENDER_PATH" "$@"

Entorno gráfico.
● ● ● bash 
services.displayManager.sddm.enable = true;
services.desktopManager.plasma6.enable = true;

# Iconos de KDE:
users.users.usuario.packages = with pkgs; [
    kdePackages.kate
    kdePackages.filelight

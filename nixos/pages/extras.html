Herramientas adicionales para desarrolladores y usuarios avanzados: nix-shell para entornos aislados, nix-ld y wrappers para binarios externos, y más.

nix-shell — Entornos aislados

¿Qué es nix-shell? Crea un entorno temporal con las dependencias que necesites. Es como un virtualenv de Python pero para cualquier lenguaje o herramienta. Dentro del shell tienes acceso a esos paquetes; afuera, no existen.

Es ideal para: no ensuciar el sistema con dependencias de un proyecto, probar herramientas rápido, o tener versiones distintas de un mismo paquete según el proyecto.

Entorno simpleCrea un shell con múltiples paquetes disponibles temporalmente.
Uso básico.
● ● ● bash 
nix-shell -p python3 nodejs gcc

Dentro del shell puedes usar python3, node, gcc. Al escribir exit, desaparecen.

Ejecutar comando y salirEjecuta un comando y sale automáticamente. Útil para scripts.
Una línea.
● ● ● bash 
nix-shell -p python3 --run "python --version"
Shell con flake.nixCrea un shell reusable basado en un archivo de configuración.mkShell permite definir dependencias arbitrarias.
Para proyectos.
● ● ● bash 
{ pkgs ? import <nixpkgs> {} }:

pkgs.mkShell {
    buildInputs = [
        pkgs.python3
        pkgs.python3Packages.pip
        pkgs.nodejs
    ];
}

Luego simplemente ejecuta nix-shell en esa carpeta.

nix-shell vs venv/pip/venv de Node: nix-shell maneja cualquier dependencia (compiladores, librerías C, herramientas del sistema), no solo paquetes del lenguaje. Es un entorno completo del sistema, no solo de paquetes.

nix-ld — Binarios externos

¿Qué es nix-ld? Permite ejecutar binarios que no fueron compilados para NixOS (como Blender descargado de la web). Proporciona las librerías del sistema que estos programas esperan encontrar en /lib o /usr/lib.

Habilitar nix-ldActiva el wrapper global para binarios externos. Añade las librerías que necesites.
Configuración básica.
● ● ● bash 
programs.nix-ld = {
    enable = true;
    libraries = with pkgs; [
        libx11 libxext libxrender
        libglvnd fontconfig freetype
        libjpeg_turbo libpng zlib
        libxkbcommon wayland
        # Para Blender con CUDA:
        linuxPackages.nvidia_x11
        cudaPackages.cuda_cudart
    ];
};

¿Cuándo usar nix-ld? Cuando descargas un binario precompilado de su web oficial (Blender, OBS, Spotify, Discord, Steam) y no hay una versión en nixpkgs o necesitas la última versión.

Wrappers con mkWrapper

Wrappers son "envoltorios" que ejecutan un binario con las librerías de Nix. Es más flexible que nix-ld porque puedes tener wrappers distintos por aplicación.

Crear wrapper manualCrea un script que establece LD_LIBRARY_PATH y ejecuta el binario.
Personalizado.
● ● ● bash 
pkgs.writeShellScriptBin "mi-app" ''
    export LD_LIBRARY_PATH=${pkgs.libgl}/lib:$LD_LIBRARY_PATH
    exec /ruta/a/binario "$@"
  ''

¿nix-ld vs mkWrapper? nix-ld es global (todos los binarios lo usan). mkWrapper es por-app. Para apps que descargas de internet y quieres que funcione "solo", usa nix-ld. Para personalización fina, usa wrappers.

Steam y Proton

Ejecutar con steam-runEnvuelve el ejecutable del juego con las librerías de Steam.
Para juegos de Windows.
● ● ● bash 
steam-run /ruta/al/juego.exe
Steam integradoHabilita Steam como programa del sistema con soporte para remote play.
Desde nixpkgs.
● ● ● bash 
programs.steam = {
    enable = true;
    remotePlay.openFirewall = true;
};
programs.nix-ld.libraries = with pkgs; [
    steam
];

Optimizar arranque de Plasma

¿El panel de Plasma tarda en aparecer? Cada widget (reloj, lanzador, notificaciones) tiene un valor PreloadWeight que determina cuándo carga. Valores altos = carga inmediata = tarda más en arrancar.

Reducir este valor hace que el panel aparezca primero y los widgets carguen en segundo plano.

Reducir PreloadWeightEdita el archivo de configuración de Plasma para acelerar el arranque.
Aplicable a KDE Plasma.
● ● ● bash 
cp ~/.config/plasma-org.kde.plasma.desktop-appletsrc ~/.config/plasma-org.kde.plasma.desktop-appletsrc.backup

Antes de modificar, haz backup del archivo.

Valores óptimosCambia estos valores en [Containments][2][Applets][X][Configuration].
Reduce de 100 a 30.
● ● ● bash 
# En ~/.config/plasma-org.kde.plasma.desktop-appletsrc:
DigitalClock PreloadWeight=100  → 30
AndromedaLauncher PreloadWeight=100 → 30
Notificaciones PreloadWeight=100 → 30
Clipboard PreloadWeight=100 → 30
SystemMonitor PreloadWeight=20   ← (opcional, menor=prioridad baja)

Busca PreloadWeight en el archivo y cambia los valores. Cierra sesión y vuelve a entrar para probar.

¿Funciona? El panel debería aparecer casi instantáneamente (~1-3s) en lugar de esperar 5-10s a que carguen todos los widgets.

Más info: Los valores van de 0 (Nunca cargar) a 100 (Cargar primero). 30 es un buen equilibrio: carga rápido pero los widgets aparecen enseguida.

Restaurar si hay problemas

Si algo sale mal, restaurar el backup es instantáneo.

Restaurar backupDeshace todos los cambios realizados.
Recuperar configuración anterior.
● ● ● bash 
cp ~/.config/plasma-org.kde.plasma.desktop-appletsrc.backup ~/.config/plasma-org.kde.plasma.desktop-appletsrc

Cierra sesión y vuelve a entrar. Si el panel no carga bien, reinicia el PC.

Consejo: Después de cada cambio significativo, renombra el backup (ej: .backup-v1, .backup-v2) para poder volver atrás si es necesario.

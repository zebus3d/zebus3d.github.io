🚀 Primeros pasos en NixOS

¿Qué es NixOS? Es una distribución de Linux donde todo se configura desde un archivo: servicios, paquetes, usuarios, red, bootloader... No usas comandos como apt install ni modificas archivos a mano. Escribes tu configuración y NixOS la hace realidad.

Esto se llama modelo declarativo: escribes lo que quieres y NixOS se encarga de traerte lo que pediste. Es como escribir la carta a los Reyes Magos: solo escribes tus deseos y alguien se encarga de traerte todo.

¿Qué es nixpkgs? Es el repositorio gigante con miles de programas para Nix/NixOS. Cuando dices pkgs.firefox, está diciendo "del repositorio nixpkgs, tráeme Firefox". Puedes buscar paquetes en search.nixos.org/packages.

Conceptos fundamentales

El Nix Store — /nix/storeEs la carpeta donde Nix guarda TODOS los archivos del sistema: programas, librerías, configuraciones. Cada versión de cada paquete tiene su propia carpeta con un hash único.
Donde Nix guarda todo. Cada paquete/version tiene su carpeta única.
● ● ● 
# Ejemplo:
/nix/store/abc123-firefox-120.0/      # Firefox
/nix/store/def456-glibc-2.38/        # Librería C
configuration.nix — tu configuraciónEl archivo principal. Aquí defines TODO: paquetes, servicios, usuarios, red, bootloader. Está en /etc/nixos/configuration.nix
El archivo principal del sistema. Editas → reconstruyes → listo.
● ● ● 
# Editar:
sudo vim /etc/nixos/configuration.nix
# Aplicar:
sudo nixos-rebuild switch
pkgs — la colección de paquetesEs la variable que contiene todos los paquetes disponibles. Cuando escribes pkgs.firefox, dices "del repositorio, quiero firefox".
pkgs es una variable con miles de programas.
● ● ● 
environment.systemPackages = with pkgs; [ firefox git vlc ];
with pkgs; es un atajo: en lugar de escribir pkgs.firefox pkgs.git, solo escribes firefox git
Perfil y generaciones — tu sistema versionadoUn perfil apunta a la generación actual. Cada rebuild crea una nueva generación. Podés volver atrás (rollback).
Cada rebuild crea una "generación". Podés volver atrás si algo falla.
● ● ● 
# Ver generaciones:
sudo nix-env -p /nix/var/nix/profiles/system --list-generations
# Volver atrás:
sudo nixos-rebuild switch --rollback

El flujo de trabajo

  • Editas tu configuración
    sudo vim /etc/nixos/configuration.nix
  • Reconstruyes el sistema
    sudo nixos-rebuild switch
  • Listo — tu sistema refleja la config
    Si algo falla: --rollback te salva

    • Editor recomendado: Usa vim si empiezas. Ejemplo: sudo vim /etc/nixos/configuration.nix. Para salir: Esc, luego :wq, luego Enter.

    💡 Dato importante: El archivo /etc/nixos/hardware-configuration.nix se genera automáticamente y contiene la config de tu hardware (discos, particiones, etc.). No lo edites a mano, se regenera cada vez que ejecutas nixos-generate-config. Tu config principal solo necesita tener imports = [ ./hardware-configuration.nix ]; al principio.

    Comandos esenciales

    ComandoPara qué
    sudo vim /etc/nixos/configuration.nixEditar la configuración
    sudo nixos-rebuild switchAplicar los cambios
    sudo nixos-rebuild switch --rollbackVolver a la generación anterior
    sudo nix-collect-garbageLiberar espacio de paquetes huérfanos

    Sobre Flakes: Esta guía cubre canales e híbrido (fetchTarball) que son más simples para empezar.

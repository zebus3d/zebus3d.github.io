🔀 Configuración Híbrida

¿Qué es una config híbrida? Es un punto medio entre canales y flakes. Usas canales para el sistema base pero importas paquetes inestables con fetchTarball sin necesidad de flakes. Es lo que usa tu config actual.

¿Por qué? Algunos paquetes (Neovim, Brave, Git, etc.) se actualizan más rápido en unstable. Con este método tienes un sistema estable pero apps siempre al día.

Ventaja: Tienes un sistema/base estable con las apps que quieras actualizadas.

Cómo usar paquetes inestables

Declarar repositorio inestableImporta nixpkgs-unstable sin crear un canal adicional. fetchTarball descarga la expresión nix actual.Se actualiza cada rebuild
Agregar nixpkgs-unstable como variable.
● ● ● 
let
  unstable = import (fetchTarball "https://nixos.org/channels/nixpkgs-unstable/nixexprs.tar.xz") {
      config.allowUnfree = true;
  };
in
Usar paquetes mezcladosPuedes mezclar paquetes de stable (pkgs) y unstable en el mismo usuario.
Ejemplo de configuración mixta.
● ● ● 
users.users.zebus3d = {
    packages = with pkgs; [
        kdePackages.kate          # ◀ estable
        unstable.neovim           # ◀ inestable
        unstable.brave            # ◀ inestable
        unstable.git              # ◀ inestable
        direnv                     # ◀ estable
    ];
};

fetchTarball vs flakes: fetchTarball descarga cada vezCada nixos-rebuild verifica si hay nueva versión. Más lento pero siempre actual. Flakes usa flake.lock para versiones fijas. — no congela versiones. Si quieres reproducibilidad total, migra a flakes.

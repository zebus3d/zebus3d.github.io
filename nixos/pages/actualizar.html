🔄 Actualizar el sistema

¿Qué significa actualizar en NixOS? A diferencia de otras distros donde "actualizar paquetes" es un comando separado, aquí actualizar significa: (1) descargar las últimas versiones del canal o flake, y (2) reconstruir todo el sistema con esas versiones. Kernel, apps, librerías — todo se actualiza de golpe.

¿Cada cuánto? Cada 1-2 semanas es lo ideal. Diario genera demasiadas generaciones. Esperar meses aumenta el riesgo de romper algo. Usuario casual ▶ una vez al mes.

Con canales

Actualizar con --upgradeHace el update del canal y rebuild en un paso. El método más sencillo.
Método rápido.
● ● ● bash 
sudo nixos-rebuild switch --upgrade
Actualizar en dos pasosPrimero actualiza el canal, luego reconstruye. Te permite verificar cambios entre pasos.
Método manual.
● ● ● bash 
sudo nix-channel --update
sudo nixos-rebuild switch

Con Flakes

Actualizar TODOS los inputsActualiza todas las dependencias según flake.nix. Equivalente a actualizar repositorios en Arch (pacman -Sy).
Actualiza las versiones de nixpkgs en flake.lock.
● ● ● bash 
cd /etc/nixos && sudo nix flake update
Actualizar solo un input específicoActualiza solo nixpkgs-unstable sin tocar nixpkgs estable. Más seguro.
Para actualizar solo los paquetes unstable.
● ● ● bash 
cd /etc/nixos && sudo nix flake lock --update-input nixpkgs-unstable
Puedes actualizar varios a la vez: --update-input nixpkgs --update-input nixpkgs-unstable
Ver versión actual de los inputsMuestra las versiones exactas que están en flake.lock sin actualizarlas.
Consulta las versiones lockeadas.
● ● ● bash 
nix flake metadata /etc/nixos
Muestra cada input con su fecha (ej: 2026-04-15) y commit hash. No requiere sudo.
Reconstruir con Flake (Opción A — con cd)Aplica la configuración del flake entrando primero al directorio.
Aplicar cambios desde /etc/nixos.
● ● ● bash 
cd /etc/nixos && sudo nixos-rebuild switch
Reconstruir con Flake (Opción B — ruta absoluta)Usa --flake /etc/nixos desde cualquier directorio. No necesitas saber el hostname.
Aplicar cambios sin cd.
● ● ● bash 
sudo nixos-rebuild switch --flake /etc/nixos
Funciona desde cualquier directorio. Solo necesitas la ruta donde está tu flake.nix.

Recordatorio: nix flake update debe ejecutarse desde /etc/nixos (haz cd primero). Si lo ejecutas desde otro lado dará: error: could not find a flake.nix file. En cambio, nixos-rebuild switch --flake /etc/nixos sí acepta ruta absoluta.

⚠️ Diferencia clave: nix flake update vs nixos-rebuild switch

nix flake update solo actualiza el flake.lock (las versiones de nixpkgs, unstable, etc.). No reconstruye el sistema. Los cambios no se aplican hasta que hagas rebuild.

nixos-rebuild switch --flake . reconstruye y aplica la configuración usando las versiones que están en el flake.lock en ese momento. Si no ejecutaste nix flake update antes, usará las versiones viejas.

Flujo correcto para actualizar todo:

● ● ● bash 
cd /etc/nixos && sudo nix flake update && sudo nixos-rebuild switch --flake .

El && encadena los comandos: si falla el update, no se ejecuta el rebuild. Así evitas aplicar cambios con un lock corrupto.

🔍 Explicación para novatos: El flake.lock es como una lista de la compra donde apuntas "nixpkgs versión X", "unstable versión Y". nix flake update es actualizar la lista con las últimas versiones disponibles. nixos-rebuild switch es ir a comprar usando lo que pone en la lista. Si no actualizas la lista primero, comprarás siempre lo mismo (las versiones viejas). Por eso hacen falta los dos pasos: primero decides qué quieres comprar (update), luego vas y lo compras (switch).

¿Reiniciar después de actualizar el kernel? Sí, el kernel se carga durante el arranque. Después de nixos-rebuild switch, reinicia para usar la nueva versión.

Verificar kernels:

● ● ● bash 
uname -r                          # Kernel actual (en ejecución)
ls -la /run/booted-system/kernel  # Kernel que se cargará en el próximo reinicio

Comandos: sudo reboot para reiniciar, o sudo nixos-rebuild boot para solo configurar kernel sin aplicar cambios inmediatos.

🔗 Montar NAS por SMB/CIFS

Montar un NAS (o cualquier carpeta compartida en red) con protocolo SMB (Samba) en NixOS se hace de forma declarativa, es decir, escribes la configuración en tu configuration.nix y NixOS se encarga de todo. Nada de editar /etc/fstab manualmente.

Esta guía asume que tienes un NAS en tu red local (por ejemplo un QNAP, Synology, o un Raspberry Pi con Samba) y quieres montar una carpeta compartida en tu sistema NixOS.

Qué necesitas saber antes de empezar

Para montar un NAS necesitas estos 4 datos. Consíguelos antes de empezar:

  • IP del NAS — Ej: 192.168.1.100. La averiguas desde el router o desde el panel de tu NAS.
  • Nombre del recurso compartido — Es el nombre que le pusiste a la carpeta en el NAS. Ej: nfs, Multimedia, Public.
  • Punto de montaje local — La carpeta local donde quieres ver los archivos del NAS. Ej: /mnt/qnap.
  • Usuario y contraseña del NAS (los que usas para conectarte desde Windows o desde tu móvil).

Paso 1: Añadir la configuración en configuration.nix

Edita tu /etc/nixos/configuration.nix con sudo (o con pkexec si usas KDE). Añade estas líneas en cualquier parte del archivo, pero tiene su lógica:

  • boot.supportedFilesystems = [ "cifs" ]; — Habilita el soporte de SMB/CIFS en el kernel de NixOS.
  • fileSystems."/mnt/qnap" = { ... }; — Declara el punto de montaje del NAS.

Explicación de cada opción

Configuración completa del montaje SMB
Copia esto en tu configuration.nix y cambia los valores según tu NAS.
● ● ● nix 
# Habilita soporte SMB/CIFS en el kernel
boot.supportedFilesystems = [ "cifs" ];

# Montaje del NAS
fileSystems"/mnt/qnap" = {
    device = "//192.168.1.100/nfs";  # ← CAMBIA: IP y nombre del share
    fsType = "cifs";
    options = let
        creds = "/etc/nixos/.smb-creds";  # Ruta del archivo de credenciales (persistente)
    in [
        "credentials=${creds}"   # Usa el archivo para autenticarse
        "vers=3.0"             # Versión del protocolo SMB (3 es la moderna)
        "uid=1000"             # Tu UID (normalmente 1000)
        "gid=100"              # Tu GID (normalmente 100)
        "iocharset=utf8"      # Codificación de caracteres
        "noperm"               # Ignora permisos del NAS (más flexible)
        "_netdev"              # Es un dispositivo de red
        "x-systemd.automount"  # Monta solo cuando accedes a la carpeta
        "noauto"               # No montar al arranque (lo hace automount)
    ];
};

Cambia //192.168.1.100/nfs por la IP y nombre de tu recurso compartido. También ajusta uid y gid si tu usuario no es el 1000 (averígualo con id).

Montaje automático vs. bajo demanda

Montaje al arrancar (automático)
Reemplaza las opciones de montaje por estas:
● ● ● nix 
"credentials=${creds}"
"vers=3.0"
"uid=1000"
"gid=100"
"iocharset=utf8"
"noperm"
"_netdev"
"auto"      # ← Monta automáticamente al iniciar sesión

Si el NAS no está encendido al arrancar, el sistema puede tardar varios segundos en intentar montarlo y fallar. El montaje bajo demanda (x-systemd.automount) evita este problema.

Paso 2: Crear el archivo de credenciales

Las contraseñas nunca se escriben en el configuration.nix porque ese archivo termina en el store de Nix (accesible globalmente en /nix/store/...). En su lugar, se guardan en un archivo externo con permisos restringidos.

Para que las credenciales sean persistentes (no se pierdan al reiniciar), las guardamos en /etc/nixos/.smb-creds. El nombre empieza con punto para que esté oculto, y aunque está en la misma carpeta que configuration.nix, el store de Nix no lo incluye porque no está referenciado en ninguna expresión Nix.

Crear el archivo .smb-creds
Ábrelo con nvim (o tu editor favorito) con sudo:
● ● ● bash 
sudo nvim /etc/nixos/.smb-creds

Escribe las credenciales, guarda (:wq) y listo. Así no queda rastro en el historial del shell.

Contenido del archivo .smb-creds
Dentro del archivo, escribe esto con tus datos:
● ● ● bash 
username=tu_usuario_del_nas
password=tu_contraseña
domain=WORKGROUP
Proteger el archivo
Solo root puede leerlo y NixOS lo ignora por estar oculto:
● ● ● bash 
sudo chmod 600 /etc/nixos/.smb-creds

El archivo empieza con punto (.smb-creds) para que nixos-rebuild no intente procesarlo. Puedes comprobarlo con ls -la /etc/nixos/.

Paso 3: Aplicar los cambios

Una vez que tienes el configuration.nix editado y el archivo de credenciales creado, toca aplicar la configuración. Si usas flakes (como es el caso aquí), el comando es el mismo:

Reconstruir el sistema
Aplica la nueva configuración de NixOS:
● ● ● bash 
cd /etc/nixos && sudo nixos-rebuild switch

El cd /etc/nixos asegura que Nix encuentre el flake.nix. Si tu flake está en otra ruta, adáptalo.

Paso 4: Probar el montaje

Con la configuración de x-systemd.automount, el NAS se monta automáticamente cuando intentas acceder a la carpeta. No necesitas montarlo manualmente.

Acceder al NAS
Simplemente entra en la carpeta:
● ● ● bash 
ls /mnt/qnap

Deberías ver los archivos de tu NAS. La primera vez tardará un segundo en montarse.

Verificar el estado del montaje
Comprueba que está montado correctamente:
● ● ● bash 
mount | grep cifs

Si ves una línea con //192.168.1.100/nfs y /mnt/qnap, está montado correctamente.

Solución de problemas

❌ "mount error(2): No such file or directory"
La IP o el nombre del recurso compartido no existen. Verifica:
  • Que la IP del NAS sea correcta (ping 192.168.1.100)
  • Que el nombre del share sea exacto (mayúsculas/minúsculas)
  • Que el NAS esté encendido y accesible en la red
❌ "mount error(13): Permission denied"
Las credenciales son incorrectas. Verifica:
  • Usuario y contraseña en /etc/nixos/.smb-creds
  • Permisos del archivo (chmod 600)
  • Prueba a conectar desde otro dispositivo para confirmar que las credenciales funcionan
❌ "mount error(112): Host is down"
El NAS no responde. Verifica:
  • Conexión de red (cable, WiFi)
  • Que el NAS esté encendido
  • Cortafuegos en el NAS (puerto 445 para SMB)

Notas importantes

¿Por qué no se usa /etc/fstab en NixOS?

En NixOS todo se configura de forma declarativa. Si editas /etc/fstab manualmente, tus cambios se perderán la próxima vez que ejecutes nixos-rebuild switch. La forma correcta es usar fileSystems en configuration.nix.

¿SMB3 es seguro?

SMB 3.0 es la versión moderna del protocolo e incluye cifrado de extremo a extremo. En redes locales domésticas es más que suficiente. Si tu NAS es muy antiguo, puede que solo soporte SMB 1.0 (inseguro). En ese caso cambia "vers=3.0" por "vers=1.0".

Alternativa más avanzada: agenix / sops-nix

Si quieres gestionar las credenciales de forma totalmente declarativa (sin tener archivos sueltos), puedes usar agenix o sops-nix. Cifran los secrets y los descifran durante la construcción. Es más complejo pero más elegante.

🔧 Bootloader

¿Qué es el bootloader? Es el menú que ves al encender el ordenador donde eliges qué generación de NixOS arrancar. Cada generación crea una entrada en ese menú.

Cuando borras generaciones, los archivos de esas entradas desaparecen pero las entradas en el menú pueden quedar huérfanas. Esta sección es para limpiarlas.

⚠️ Cuidado con Plasma

Limpiar generaciones + GC puede romper los iconos anclados de Plasma.

Los launchers guardan rutas absolutas del Nix Store. Al ejecutar GC, esas rutas desaparecen.

  • Afecta a TODAS las apps ancladas (Discord, Telegram, KeePassXC...)
  • La solución: editar ~/.config/plasma-org.kde.plasma.desktop-appletsrc y usar applications:nombre.desktop

Limpiar entradas huérfanas

Método principal: Ejecutar nixos-rebuild boot reconstruye el menú del bootloader con solo las generaciones válidas.

Limitar entradas

Limitación de generaciones en systemd-boot: Para limitar el número de entradas en el menú de arranque a 3-4, añade esta opción en configuration.nix:

● ● ● bash 
bootloadersystemd-bootconfigurationLimit = 3;

Las entradas reported/absent en bootctl list son variables EFI inofensivas que se borran al reiniciar. nixos-rebuild boot reconstruye el menú desde cero con solo las generaciones válidas.

Añadir Windows 11 a systemd-boot

¿Qué es una ESP? ESP son las siglas de EFI System Partition. Es una partición formateada en FAT32 donde el firmware UEFI busca los archivos del bootloader (archivos .efi). En este ordenador hay dos ESPs separadas: nvme0n1p1 (la de Windows) y nvme0n1p5 (la de NixOS, montada en /boot). Tenerlas separadas significa que systemd-boot no ve a Windows automáticamente — hay que copiarle sus archivos EFI manualmente.

⚠️ Particiones — Este procedimiento asume que Windows tiene su propia ESP (nvme0n1p1) separada de la de NixOS (nvme0n1p5). Ajusta los números de partición según tu disco.

Método: copiar los EFI de Windows a /boot/ — systemd-boot detecta automáticamente entradas Windows si los archivos EFI están en /boot/EFI/Microsoft/. No es necesario usar boot.loader.systemd-boot.extraEntries (eso causaría una entrada duplicada).

Copiar EFI de Windows a /boot/Monta la ESP de Windows, copia los archivos EFI y desmonta. systemd-boot auto-detecta Windows automáticamente.Hacer tras instalar/actualizar Windows
Habilitar entrada de Windows en systemd-boot.
● ● ● bash 
sudo mount /dev/nvme0n1p1 /mnt
sudo cp -r /mnt/EFI/Microsoft /boot/EFI/
sudo cp -r /mnt/EFI/Boot /boot/EFI/
sudo umount /mnt
⚠️ Trampas comunes
  • NO uses extraEntries en configuration.nix para Windows — al copiar los EFI, systemd-boot auto-detecta y genera una entrada. Si además pones extraEntries, aparecen dos entradas de Windows en el menú.
  • Un nixos-rebuild puede sobrescribir /boot/ y borrar los EFI copiados. Si tras un rebuild Windows desaparece del menú, re-ejecuta los comandos de copia.
  • Si en el pasado creaste un archivo manual /boot/loader/entries/windows.conf, ese archivo no se borra solo con nixos-rebuild porque fue creado fuera de NixOS. Si alguna vez aparece Windows duplicado, revísalo y bórralo manualmente.

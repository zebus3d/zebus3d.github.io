🗑️ 1. Eliminar generaciones antiguas

Método recomendado — Todo en uno

✓ Recomendado
Borrar generaciones de más de 30 días + Garbage CollectionElimina generaciones con más de 30 días del perfil del sistema y ejecuta GC automáticamente. Es seguro y conservador. Ideal para uso semanal.Ej: si hiciste rebuild hace 35 días, esa generación se borra
Elimina generaciones antiguas de todo el equipo y ejecuta el recolector de basura automáticamente.
bash 
sudo nix-collect-garbage --delete-older-than 30d
✓ Recomendado
Limpieza profunda — Todas las antiguas de todos los usuariosRecorre todos los perfiles del sistema (usuario, root, system) y elimina todas las generaciones antiguas. Usa con precaución si necesitas rollback a versiones muy antiguas.Ej: después de actualizar y confirmar que todo funciona bien
Borra TODAS las generaciones antiguas de todos los perfiles (sistema + usuario + root). Es el más efectivo para ganar espacio rápidamente.
bash 
sudo nix-collect-garbage -d

Método por perfil — Control manual

Ver generaciones del perfil de sistemaMuestra todas las generaciones del perfil del sistema con fecha, número y si es la actual. Útil para saber qué vas a borrar antes de ejecutar nada.
bash 
sudo nix-env -p /nix/var/nix/profiles/system --list-generations
Eliminar todas excepto las últimas 5Mantiene las 5 generaciones más recientes y borra el resto. Útil si quieres un rollback seguro a las últimas versiones sin acumular generaciones viejas.Ej: si tienes 20 generaciones, borra 15 y deja 60-64
bash 
sudo nix-env -p /nix/var/nix/profiles/system --delete-generations +5
Eliminar las de más de 30 días solo en el sistemaBorra generaciones del perfil system con más de 30 días. Más conservador que `+5`, respeta las recientes. Ideal si haces rebuilds frecuentes y quieres mantener un historial de un mes.
bash 
sudo nix-env -p /nix/var/nix/profiles/system --delete-generations 30d

Seguridad al eliminar generaciones

La generación actual NUNCA se elimina. NixOS la protege automáticamente.

  • Los comandos de borrado solo quitan los índices. Los archivos reales del /nix/store solo desaparecen tras ejecutar el Garbage Collector.
  • IMPORTANTE: sudo nix-collect-garbage -d borra las generaciones de TODOS los perfiles (sistema + tu usuario + root). Es el más efectivo para ganar espacio rápidamente.

Orden de riesgo

Nivel Comando Descripción
SEGURO --delete-generations 30d Conserva el último mes
MODERADO --delete-generations +5 Conserva las 5 últimas
AGRESIVO --delete-generations old Solo deja la actual
MUY AGRESIVO nix-collect-garbage -d Todo en uno, sin red de seguridad

Flujo recomendado para limpiar con seguridad

  1. Reconstruir el sistema PRIMERO (evita que se rompan iconos de Plasma)
    bash 
    sudo nixos-rebuild switch

    Actualiza las rutas del system profile para que los launchers de Plasma no pierdan sus referencias

  2. Listar generaciones para ver qué hay
    bash 
    sudo nix-env -p /nix/var/nix/profiles/system --list-generations
  3. Eliminar solo lo antiguo (30d es bastante conservador)
    bash 
    sudo nix-env -p /nix/var/nix/profiles/system --delete-generations 30d
  4. Ejecutar el Garbage Collector
    bash 
    sudo nix-collect-garbage
  5. Regenerar el bootloader desde cero
    bash 
    sudo nixos-rebuild boot

    Reconstruye el menú de arranque con solo las generaciones vigentes. Más seguro que bootctl cleanup, no deja entradas fantasma

💾 Liberar espacio

IMPORTANTE: No tener procesos de compilación o descarga activos al ejecutar estos comandos.

✓ Recomendado
Borrar archivos no usados del /nix/storeElimina los archivos reales del store que ya no tienen referencias. Es el que realmente libera espacio en disco. Seguro, no borra nada en uso.Ej: después de borrar generaciones, libera GB reales
El borrado de generaciones solo quita los "punteros". Este comando borra los archivos reales.
bash 
sudo nix-collect-garbage
Todo en uno — Borrar generaciones antiguas + GCCombina --delete-older-than y garbage collection en un solo comando. Borra generaciones viejas de todos los perfiles y elimina los archivos reales. Es el más práctico para uso frecuente.Ej: sudo nix-collect-garbage -d (borra todo lo que no tiene referencia)
bash 
sudo nix-collect-garbage -d
✓ Recomendado
Optimizar el store — Deduplicar archivos idénticosSi dos paquetes comparten archivos iguales, crea hardlinks para que ocupen una sola vez. No borra nada, solo optimiza. Ideal ejecutar tras cada GC.Ej: si tienes 3 versiones de glibc, solo ocupa una
bash 
sudo nix-store --optimise

🔧 Limpiar entradas huérfanas del bootloader

⚠️ ADVERTENCIA CRÍTICA — Leer antes de limpiar ⚠️

La combinación de nix-collect-garbage + bootctl cleanup puede romper los iconos anclados de Plasma Desktop.

Los launchers de Plasma guardan rutas absolutas del Nix Store a los archivos .desktop. Al ejecutar nix-collect-garbage, esas rutas se eliminan y los iconos desaparecen de la barra de tareas.

  • Esto afecta a TODAS las apps ancladas (Discord, Telegram, KeePassXC, etc.)
  • Los iconos del sistema (Kate, Plasma System Monitor) pueden sobrevivir si están en el system profile
  • La solución requiere editar manualmente ~/.config/plasma-org.kde.plasma.desktop-appletsrc y usar el formato applications:nombre.desktop

Orden seguro de operaciones (SIEMPRE en este orden):

1. sudo nixos-rebuild switch — Reconstruir el sistema primero
2. sudo nix-collect-garbage -d — Luego limpiar generaciones
3. sudo bootctl cleanup — Finalmente limpiar el bootloader

NUNCA ejecutes nix-collect-garbage sin haber hecho nixos-rebuild switch antes. Si limpias el store antes de reconstruir, las rutas que Plasma tiene guardadas desaparecen.

Eliminadas las generaciones, los archivos .conf en /boot/loader/entries/ quedan huérfanos. Hay dos formas de limpiarlos:

Opción A — Solo limpiar entradas huérfanas (más rápido)Elimina archivos .conf del bootloader cuya generación ya no existe. Rápido pero puede dejar entradas "fantasma" en la memoria EFI que se limpian solas al reiniciar.
⚡ Funciona bien, pero a veces deja entradas "fantasma" en la memoria EFI. Si ves entradas reported/absent, reinicia el equipo.
bash 
sudo bootctl cleanup
✓ Recomendado
Opción B — Regenerar el bootloader completo desde ceroReconstruye todas las entradas del menú de arranque desde cero. Es más seguro que bootctl cleanup porque no depende de archivos huérfanos. No deja entradas fantasma en la EFI.Ej: si tenías 5 generaciones válidas, el menú solo muestra esas 5
Reconstruye el menú de arranque desde cero con solo las generaciones vigentes. Es la forma más limpia y segura. No deja entradas fantasma.
bash 
sudo nixos-rebuild boot

Con nixos-rebuild boot no es necesario bootctl clean, hace todo junto. Usa bootctl clean si ya corriste nix-collect-garbage y solo quieres limpiar el menú.

Recomendación segura: En lugar de usar bootctl cleanup manualmente, es más seguro usar sudo nixos-rebuild boot que regenera el bootloader desde cero con solo las generaciones vigentes. Evita cualquier efecto secundario con rutas huérfanas en el escritorio.

Notas sobre entradas fantasma (reported/absent): A veces bootctl list muestra entradas antiguas marcadas como reported/absent. Son variables EFI almacenadas en la memoria del firmware, no son archivos. Son inofensivas y se eliminan solas al reiniciar el equipo. No intentes borrarlas manualmente.

🔄 2. Actualizar el sistema

Usando Canales (Standard)

Actualizar canales y aplicar configuraciónDescarga la última versión del canal de NixOS y aplica los cambios. Es el método en dos pasos, útil si quieres revisar qué ha cambiado antes de aplicar.Ej: después de modificar configuration.nix, ejecuta estos dos comandos
bash 
sudo nix-channel --update
sudo nixos-rebuild switch
Actualizar y aplicar en un solo pasoCombina la actualización del canal y la reconstrucción en un solo comando. Más rápido, pero no te permite revisar los cambios antes de aplicarlos.Ej: sudo nixos-rebuild switch --upgrade (actualiza y aplica todo)
bash 
sudo nixos-rebuild switch --upgrade

Usando Flakes

1. Actualizar flake.lockActualiza todas las dependencias del flake a sus últimas versiones y regenera el archivo flake.lock. Es el equivalente a hacer nix-channel --update pero con flakes.Ej: nix flake update (actualiza todo a las últimas versiones)
bash 
nix flake update
2. Aplicar configuraciónConstruye y activa la nueva configuración del sistema usando el flake. El `.#` indica que use la configuración del directorio actual con el nombre del host por defecto.Ej: sudo nixos-rebuild switch --flake .#mi-pc
bash 
sudo nixos-rebuild switch --flake .

Si usas un nombre de host específico:

bash 
sudo nixos-rebuild switch --flake .#nombre-de-host

⚙️ 3. Automatización (configuration.nix)

Limpieza automática periódicaConfigura un timer de systemd que ejecuta nix-collect-garbage automáticamente. "weekly" = una vez por semana. "--delete-older-than 30d" = solo borra generaciones con más de 30 días.Ej: cada domingo limpia generaciones viejas sin que hagas nada
nix 
nix.gc = {
    automatic = true;
    dates = "weekly";
    options = "--delete-older-than 30d";
};
Optimización automática (ahorra mucho espacio)Cada vez que se añade algo al store, Nix deduplica automáticamente los archivos idénticos mediante hardlinks. Sin intervención manual, sin riesgo.Ej: si instalas Firefox y ya tenías una versión similar, solo ocupa la diferencia
nix 
nix.settings.auto-optimise-store = true;
Limitar generaciones en el menú de arranque (systemd-boot)Systemd-boot solo mostrará las N generaciones más recientes en el menú de arranque. No borra las generaciones, solo las oculta del menú. Evita un menú infinito.Ej: con configurationLimit = 10, solo ves las 10 últimas al arrancar
nix 
boot.loader.systemd-boot.configurationLimit = 10;

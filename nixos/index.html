🧹 Garbage Collector

¿Qué es el Garbage Collector? En NixOS nada se borra automáticamente. Cada vez que actualizas el sistema, se crea una nueva generación (una foto completa de tu sistema) y todos los archivos se guardan en /nix/store. Con el tiempo esto acumula gigas de versiones que ya no usas.

El GC es el comando que libera ese espacio. Primero borras generaciones (los "punteros") y luego el GC elimina los archivos reales que ya nadie referencia. Es como vaciar la papelera de reciclaje.

La generación actual (la que estás usando ahora) NUNCA se elimina. Siempre puedes hacer rollback.

IMPORTANTE: No tener compilaciones o descargas activasAplica a: nix-collect-garbage, -d y --delete-older-than. Si nixos-rebuild está descargando/compilando y el GC borra esas referencias, falla con "path does not exist". al ejecutar el GC.

🗑️ Gestionar generaciones

¿Qué es una generación? Cada vez que ejecutas nixos-rebuild switch, NixOS crea una nueva generación: una "foto" completa de tu sistema en ese momento con todos los paquetes, configuraciones y versiones exactas.

Esto te permite volver atrás si algo se rompe. Pero cada generación ocupa espacio. Gestionar generaciones significa decidir cuántas conservar y borrar las que ya no necesitas.

Generaciones ≠ espacio liberado. Borrar generaciones solo quita los "punteros". Para liberar GB reales necesitas ejecutar el GC (pestaña GC).

Ver generaciones del sistemaMuestra todas las generaciones con fecha, número y cuál es la actual.
bash
sudo nix-env -p /nix/var/nix/profiles/system --list-generations
Eliminar excepto las últimas 5Mantiene las 5 más recientes. Útil para rollback seguro.Ej: 20 generaciones → borra 15
bash
sudo nix-env -p /nix/var/nix/profiles/system --delete-generations +5
Eliminar las de más de 30 díasBorra generaciones del perfil system con +30 días. Más conservador que +5.
bash
sudo nix-env -p /nix/var/nix/profiles/system --delete-generations 30d

Riesgo de --delete-generations

NivelComandoDescripción
SEGURO30dConserva el último mes
MODERADO+5Conserva las 5 últimas
AGRESIVOoldSolo deja la actual

Para los comandos de GC (que liberan espacio real), consulta la pestaña GC arriba.

🔧 Limpiar bootloader

¿Qué es el bootloader? Es el menú que ves al encender el PC donde eliges qué generación de NixOS arrancar. Cada generación crea una entrada en ese menú.

Cuando borras generaciones, los archivos de esas entradas desaparecen pero las entradas en el menú pueden quedar huérfanas. Esta sección es para limpiarlas.

⚠️ ADVERTENCIA — Plasma Desktop

nix-collect-garbage + bootctl cleanup puede romper los iconos anclados de Plasma.

Los launchers guardan rutas absolutas del Nix Store. Al ejecutar GC, esas rutas desaparecen.

  • Afecta a TODAS las apps ancladas (Discord, Telegram, KeePassXC...)
  • La solución: editar ~/.config/plasma-org.kde.plasma.desktop-appletsrc y usar applications:nombre.desktop

Orden seguro:

1. sudo nixos-rebuild switch
2. sudo nix-collect-garbage -d
3. sudo nixos-rebuild boot

Opción A — Limpiar entradas huérfanasElimina .conf del bootloader de generaciones que ya no existen. Puede dejar entradas "fantasma" en la EFI.
⚡ Rápido, pero a veces deja entradas reported/absent. Reinicia para limpiarlas.
bash
sudo bootctl cleanup
✓ Recomendado
Opción B — Regenerar bootloaderReconstruye el menú de arranque desde cero. No deja entradas fantasma.Ej: 5 generaciones válidas → el menú solo muestra esas 5
Reconstruye el menú desde cero con solo las generaciones vigentes.
bash
sudo nixos-rebuild boot

Con nixos-rebuild boot no hace falta bootctl cleanup. Las entradas reported/absent en bootctl list son variables EFI inofensivas que se borran al reiniciar.

❄️ Flakes

¿Qué son los Flakes? Son la forma moderna de gestionar tu configuración de NixOS. Imagina que tu configuración es un "proyecto" con sus propias dependencias versionadas, como un proyecto de programación con su package.json.

El archivo flake.nix declara qué usar y el flake.lock "congela" las versiones exactas para que la compilación sea siempre idéntica. Esto hace que tu sistema sea reproducible: si copias los archivos a otro PC, obtienes exactamente lo mismo.

Canales vs Flakes: Los canales son como un "apt update" global (simple pero menos control). Los flakes son proyectos independientes con dependencias versionadas (más control, más reproducible).

¿Uso Flakes o Canales?Si tu config tiene "nix.settings.experimental-features" con "nix-command flakes", usas Flakes.
bash
nix-channel --list

Si muestra nixos https://nixos.org/channels/... → Canales. Si no → Flakes.

Estructura típicaflake.nix define dependencias. flake.lock congela versiones. nixos/ contiene tu configuration.nix.
tree
mi-config/
├── flake.nix        # Dependencias y outputs
├── flake.lock       # Versiones congeladas
└── nixos/
    ├── configuration.nix
    └── hardware-configuration.nix
Actualizar dependenciasEquivalente a nix-channel --update pero con flakes.
bash
nix flake update
Actualizar solo un inputSolo actualiza una dependencia sin tocar las demás. Útil si algo se rompió.Ej: nix flake lock --update-input nixpkgs
bash
nix flake lock --update-input nixpkgs

🔄 Actualizar el sistema

¿Qué significa actualizar en NixOS? A diferencia de otras distros donde "actualizar paquetes" es un comando separado, aquí actualizar significa: (1) descargar las últimas versiones del canal o flake, y (2) reconstruir todo el sistema con esas versiones. Kernel, apps, librerías — todo se actualiza de golpe.

¿Cada cuánto? Cada 1-2 semanas es lo ideal. Diario genera demasiadas generaciones. Esperar meses aumenta el riesgo de romper algo. Usuario casual → una vez al mes.

Canales

Actualizar y aplicar en un pasoActualiza canal + reconstruye. Rápido pero no permite revisar cambios antes.Ej: sudo nixos-rebuild switch --upgrade
bash
sudo nixos-rebuild switch --upgrade
En dos pasos (revisar antes)Útil si quieres ver qué cambió antes de aplicar.
bash
sudo nix-channel --update
sudo nixos-rebuild switch

Flakes

1. Actualizar flake.lock
bash
nix flake update
2. Aplicar configuración
bash
sudo nixos-rebuild switch --flake .

Con host específico:

bash
sudo nixos-rebuild switch --flake .#nombre-de-host

📦 Instalar y buscar paquetes

¿Cómo se instala software en NixOS? Hay dos formas: declarativa (añades el paquete a configuration.nix y se instala al hacer rebuild) o imperativa (lo instalas al momento con nix-env). La declarativa es la recomendada porque es reproducible y persistente.

¿Dónde está mi configuración? El archivo principal es /etc/nixos/configuration.nix. Ahí se define TODO: paquetes, servicios, usuarios, red, etc.

Buscar paquetes

Buscar en la terminalBusca paquetes disponibles en tu canal o flake. Muestra nombre, versión y descripción.Ej: nix search nixpkgs firefox
bash
nix search nixpkgs firefox
Buscar en la webEl buscador oficial de paquetes de NixOS. Más cómodo que la terminal para explorar.
search.nixos.org/packages

Instalar (declarativo — recomendado)

✓ Recomendado
Añadir a configuration.nixEdita /etc/nixos/configuration.nix y añade los paquetes. Se instalan al hacer nixos-rebuild switch.Ej: environment.systemPackages = [ pkgs.firefox pkgs.git ];
Edita /etc/nixos/configuration.nix:
nix
environment.systemPackages = with pkgs; [
    firefox
    git
    vlc
];

Luego reconstruye:

bash
sudo nixos-rebuild switch

Instalar (imperativo — rápido)

Instalar para el usuario actualInstala solo para tu usuario. No requiere sudo ni rebuild. Se pierde si no está en la config.Ej: nix-env -iA nixpkgs.firefox
Rápido pero no persistente entre reinstalaciones.
bash
nix-env -iA nixpkgs.firefox
Probar un paquete sin instalarDescarga y ejecuta un paquete temporalmente sin instalarlo. No deja rastro en el sistema.Ej: nix-shell -p python3 --run "python --version"
Ejecuta algo una sola vez sin instalarlo.
bash
nix-shell -p python3 --run "python --version"

Rollback — Deshacer cambios

✓ Salva-vidas
Volver a la generación anteriorSi algo se rompió tras un rebuild, esto vuelve a la generación anterior inmediatamente. No borra nada, solo cambia a la versión previa.
¿Algo se rompió tras actualizar? Vuelve atrás:
bash
sudo nixos-rebuild switch --rollback

⚙️ Automatización

¿Por qué automatizar? NixOS acumula espacio y generaciones con cada actualización. En lugar de limpiar manualmente, puedes configurar timers que lo hagan solos. Todo se configura en /etc/nixos/configuration.nix.

Limpieza automática semanalTimer de systemd que ejecuta GC automáticamente. "weekly" = una vez por semana.Ej: cada domingo limpia generaciones viejas
nix
nix.gc = {
    automatic = true;
    dates = "weekly";
    options = "--delete-older-than 30d";
};
Optimización automática del storeDeduplica archivos idénticos automáticamente al añadir algo al store.
nix
nix.settings.auto-optimise-store = true;
Limitar generaciones en el menú de arranqueSystemd-boot solo muestra las N generaciones más recientes. No borra, solo oculta del menú.
nix
boot.loader.systemd-boot.configurationLimit = 10;
